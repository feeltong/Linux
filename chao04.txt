부트 매니저

리눅스 부팅 과정

전원ON -> ROM_BIOS 실행 -> 부트로더 실행 -> 스와퍼 프로세스 수행 -> init 프로세스 수행 -> 부팅 레벨 결정 -> rc.sysinit 스크립트 실행 -> rcX.d 스크립트 실행 -> X 윈도우 실행

1] 부트로더 기능
    : 디스크나 플래시에 저장된 운영체재를 읽어 주기억장치에 적재해 주는 프로그램
    : 임베디드 시스템 부트로더는 PC 의 BIOS 와 OS Lorder 의 기능을 수행하는 프로그램으로 시스템이 부팅할 떄 가장 먼저 수행된다.
    : 부트로더는 부트 매니저라고도 부르며 크기가 512바이트로 하드디스크의 첫 번째 섹터인 MBR 에 위치한다
    : LILO 는 리눅스 운영체제에 한정되어 사용되는 반면 GRUB은 리눅스 운영체제 외에도 다른 운영체제에서도 사용 가능

2]GRUB
    : 리눅스의 전통적인 부트로더로 사용되어 있던 LILO 의 단점을 보완하 ㄴ것
    : 부트 정보를 사용자가 임의로 변경해 부팅할 수 있어 , 부트 정보가 올바르지 않더라고 부팅시 바로 수정하여 부팅할 수 있다.
    : 다른 여러 가지 운영체제와 멀티부팅할 수 있다.


3]런레벨
    : 리눅스 부팅 시 작동하는 서비스들이 있다. 런레벨에 따라 작동하는 서비스를 조정 가능하다
    : 런레벨은 0에서부터 6까지 총 7가지이다
    : 대표적인 3(텍스트모드) 5(그래픽모드) 가 있다.
    : 런레벨 1 은 ROOT 패스워드 분실 ,파일 시스템 점검 및 복구 , 시스템 점검을 할 떄 접근(0 은 종료 )

4]로그인과 로그아웃
    : 리눅스는 X윈도우상에서의 로그인/로그아웃과 콘솔상에서의 로그인/로그아웃이 있다
    : 로그인 과정을 보자!
    ㄴ입력한 패스워드와 파일 /etc/passwd 필드를 비교한다 -> 셀 설정 파일을 실행
    ㄴ셀 성정 파일 이 하는 일 1.검색 경로 설정 2.터미널 종류 설정 및 다양한 터미널 설정 3.환경변수 설정 4.로그인 시 실행 명령어 실행 5.로그인 메시지 출력
    : 로그인 셀을 실행한다

로그아웃]
    : logout , exit 또는 조합기 ctrl + d 를 사용한다
    : 관리자는 일정시간 동안 작업을 수행하지 않는 모든 사용자들을 강제로 로그아웃할 수 있다.
    : 강제 로그아웃은 /etc/profile 에 TIMEOUT 변수에 값을 지정하여 해당 작업을 수행한다
